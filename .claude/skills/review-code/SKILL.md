# review-code

**자가승인 금지** — 구현과 다른 패스(가능하면 서브에이전트)로 리뷰한다.

## 1단계: 발견 (커버리지)

확신이 낮거나 저심각도로 보이는 것도 **전부** 보고한다. 각 이슈마다:

1. 내용 2. 영향 범위 3. 확신도(상/중/하) 4. 심각도(상/중/하) 5. 재현/검증 방법.

이 단계에서 중요도 기준 필터링 **금지**. ("중요한 것만"이라고 지시하면 4.8이 저심각도를 조용히 빼버린다.)

## 2단계: 필터

실제 사용자 문제·테스트 실패·데이터 손상·보안 리스크로 이어지는 것만 우선순위화. 순수 네이밍/스타일 취향은 제외.

## 보안 민감 영역 (필수 2단계 리뷰)

인증/세션/쿠키(`lib/auth`, `middleware.ts`), 토큰 암호화(`lib/crypto`), 채널 OAuth(`lib/*/oauth`), 결제. 이 영역 변경 시 반드시 리뷰 패스를 거친다.

## 출력

`severity(Critical/High/Medium/Low)` + `파일:라인` + 수정안. 리뷰어는 코드를 수정하지 않는다(리뷰만). 수정은 별도 단계에서 동의 후.
